# Gating-Agency Refactor - 2026-05-21

## Change

The thalamus architecture no longer reports gating activity as `agency_proxy`.

New naming:

- `gating_coordination_proxy`: information-selection / gating dynamics.
- `action_agency_proxy`: action-outcome prediction and control.
- `legacy_agency_proxy`: old mixed score retained for diagnostics.
- `agency_proxy`: now maps to `action_agency_proxy`.

The old `gating_agency_proxy` field remains as a backward-compatible alias, but should not be interpreted as agency.

## Why

The diagnostic run showed a gating-agency confusion:

- thalamus without workspace or action loop had high old agency proxy,
- independent agency tests were low,
- intermediate-variable analysis showed the old proxy was tracking gating dynamics, not action-outcome causality.

## Remaining Problem

Self-model validation is still not solved:

- Overall self-model correlation remains weak because the independent self tests and the self proxy disagree across architecture and workspace conditions.
- Distributed self-model validation is promising.
- Thalamus self-model still needs architecture-specific boundary and ownership probes.

## Interpretation

The original mechanism result is not discarded.  It is refined:

- The old high thalamus "agency" in no-action-loop systems was gating coordination.
- The true action-agency signal appears only when action-outcome loops or action feedback are present.
- This supports the method-paper claim that theoretically motivated internal proxies can silently measure the wrong construct unless validated against independent tests.
